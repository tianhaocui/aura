## 1. Aura 自动注册自身为 bean

### 修复

`start()` 里 services resolve 之前自动注册：

```java
registry.putIfAbsent(Aura.class, this);
```

之后 Service 构造器里写 `public AuthService(Aura app, Db db)` 直接拿到。

## 2. services() 里 @Path 类自动注册路由

### 动机

AI 加一个 Controller 需要改 2 处（services + routes），容易漏。@Path 注解本身就是显式声明路由前缀，放在 services() 里应该自动注册路由。

### 行为

```java
app.services(
    UserService.class,       // 无 @Path → 纯 DI
    UserController.class,    // 有 @Path("/api/users") → DI + 路由自动注册
    AuthController.class     // 有 @Path("/api/auth") → DI + 路由自动注册
)
```

### 为什么不隐式

@Path 注解写在类的第一行，是显式的路由声明。AI grep `@Path` 就能找到所有路由来源。用户显式把类列在 services() 里了，不是 classpath 扫描。

### 认证模式

不加 @Auth 注解。用路径规则集中管理：

```java
app.before(Aura.requireAuth())
   .exclude("/health", "/api/auth/login", "/api/auth/register");
```

### 实现

start() 里 services 实例化完后，检查 @Path 注解并注册路由：

```java
for (Object bean : newlyCreatedServices) {
    if (bean.getClass().isAnnotationPresent(Path.class)) {
        ServiceRegistrar.register(bean, router);
    }
}
```

### scan() 废弃

`scan()` 标记为 `@Deprecated`。services() 是唯一推荐方式。

## 3. ctx.jsonRaw(String) — 原样输出 JSON

### 动机

已经是 JSON 字符串（如从第三方 API 拿到的响应），用 `ctx.json()` 会二次序列化。目前要写两步。

### API

```java
ctx.jsonRaw(rawJsonString);  // 设 Content-Type: application/json + 直接发 body，不序列化
```

### 实现

```java
@Override public void jsonRaw(String json) {
    exchange.getResponseHeaders().put(Headers.CONTENT_TYPE, "application/json; charset=utf-8");
    exchange.getResponseSender().send(json);
}
```

## 4. app.resultWrapper() — 全局返回值包装

### 动机

50 个路由每个都写 `ctx -> ctx.json(Result.ok(service.method(ctx)))`。Service 方法直接 return 业务对象，框架自动包 Result。

### API

```java
app.resultWrapper(Result::ok);  // 一行，全局声明
```

### 行为

- handler 正常返回值 → 经过 wrapper → `Result.ok(returnValue)` → JSON 序列化
- handler 返回 null → `Result.ok(null)`
- handler 抛异常 → 走 exceptionHandler → **不过 wrapper**
- handler 里手动调了 ctx.json()/ctx.raw()/ctx.text() → **不过 wrapper**（响应已发送）

### 示例

```java
@Path("/api/users")
public class UserController {
    @Get("/")
    List<User> list() { return db.table("user").find(); }

    @Get("/{id}")
    User get(int id) { return db.findById("user", id); }
}

// 响应自动变成：
// {"code": 0, "message": "ok", "data": [...]}
// {"code": 0, "message": "ok", "data": {"name":"alice"}}
```

### 分页

```java
@Get("/")
Page<User> list(int pageNum, int pageSize) {
    return db.table("user").paginate(pageNum, pageSize);
}
// {"code":0, "data":{"list":[...],"pageNum":1,"pageSize":20,"total":100}}
```

### 实现

```java
// Aura.java
private Function<Object, Object> resultWrapper;

public Aura resultWrapper(Function<Object, Object> wrapper) {
    this.resultWrapper = wrapper;
    return this;
}
```

路由执行时：handler 返回值 → 检查响应是否已发送 → 未发送且 wrapper != null → `ctx.json(wrapper.apply(returnValue))`

### AI 唯一推荐模式（最终形态）

```java
Aura.create()
    .register(Db.create(...))
    .resultWrapper(Result::ok)
    .services(
        UserService.class,        // 纯业务
        UserController.class,     // @Path → 路由自动注册
        OrderController.class
    )
    .before(Aura.requireAuth()).exclude("/health", "/api/auth/*")
    .start();
```

App.java 只剩配置，没有路由代码。

## 语义分工

```
register(instance)        — 基础设施：已创建的实例（Db、Config Record）
services(Class...)        — 所有组件：构造器注入。有 @Path 的自动注册路由
resultWrapper(fn)         — 全局返回值包装（可选）
```

---

## 不做

- ~~controllers(Class...)~~ — 多一个 API 违反"API 少"原则
- ~~r.get(path, Class, "method")~~ — @Path 模式覆盖了
- ~~@Named~~ — 用具体类型消歧义，不引入 DI 注解
- ~~@Auth 类注解~~ — 用路径规则集中管理
