# Aura v0.6.2 PRD — 生产样板消除

## 背景

v0.6.1 完成了 services/@Path/resultWrapper 的 AI 体验优化。v0.6.2 目标是消除实际生产项目中频繁出现的样板代码。四个功能合计 ~300 行框架代码，省用户数百行重复逻辑。

## 需求总览

| # | 功能 | 类型 | 实际痛点 |

| 1 | after().exclude() | feat | 审计日志不想记 /health |
| 2 | db.findOneOrThrow() | feat | 30+ 处 if (row == null) throw notFound |
| 3 | app.rateLimit() | feat | 注册/登录/AI 接口无防刷保护 |

## 1. after().exclude()

### API

```java
app.after(ctx -> auditLog(ctx)).exclude("/health", "/metrics");
```

### 实现

把 `BeforeBuilder` 重命名为 `FilterBuilder`，before() 和 after() 共用：

```java
public FilterBuilder before(BaseHandler handler) { ... }
public FilterBuilder after(BaseHandler handler) { ... }

// FilterBuilder
public FilterBuilder exclude(String... paths) { ... }
public FilterBuilder and() { return app; }  // 回到 Aura 链
```

## 2. db.findOneOrThrow()

### 动机

生产项目写了 30+ 次：
```java
Row row = db.table("user").where("id", id).findOne();
if (row == null) throw new BizException(404, "User not found");
```

### API

```java
// Query 级别
Row row = db.table("user").where("id", id).findOneOrThrow();
// 默认抛 IllegalArgumentException("Not found: user")

// 自定义消息
Row row = db.table("user").where("id", id).findOneOrThrow("User not found");

// Db 级别快捷方法
Row row = db.findByIdOrThrow("user", id);
```

### 实现

```java
// Query.java
public Row findOneOrThrow() {
    Row row = findOne();
    if (row == null) throw new IllegalArgumentException("Not found: " + table);
    return row;
}

public Row findOneOrThrow(String message) {
    Row row = findOne();
    if (row == null) throw new IllegalArgumentException(message);
    return row;
}

// Db.java
public Row findByIdOrThrow(String table, Object id) {
    Row row = findById(table, id);
    if (row == null) throw new IllegalArgumentException("Not found: " + table + " id=" + id);
    return row;
}
```

框架默认 `IllegalArgumentException` → 400。如果用户注册了自定义 BizException → 可以自己 throw。

## 3. @RateLimit — 方法级限流

### API

```java
@Post("/login")
@RateLimit(5)  // 每 IP 每分钟 5 次（默认窗口 1 分钟）
public Token login(LoginReq req) { ... }

@Get("/recommend")
@RateLimit(value = 10, window = 60)  // 每 IP 60 秒内 10 次
public List<Item> recommend(int userId) { ... }
```

全局默认限流仍用编程式（不需要注解）：
```java
app.rateLimit(100, Duration.ofMinutes(1));  // 全局兜底
```

### 注解定义

```java
@Retention(RetentionPolicy.RUNTIME)
@Target(ElementType.METHOD)
public @interface RateLimit {
    int value();              // 最大请求数
    int window() default 60; // 时间窗口（秒）
}
```

### 实现

- MethodRefHandler 执行前检查 @RateLimit 注解
- 内存计数器 ConcurrentHashMap<IP+Method, Counter>
- 超限返回 429 + `{"error":"Rate limit exceeded","retryAfter":32}`
- 启动日志：`Rate limit: AuthController.login (5/60s)`

## 4. @Scheduled / Cron 定时任务

### 动机

成就计算、streak 重置、数据清理需要 cron。当前手写 `ScheduledExecutorService`。

### API

```java
// 声明式 — Service 方法上加注解
public class StatsService {
    @Scheduled(cron = "0 0 * * *")  // 每天凌晨
    public void resetDailyStreak() { ... }

    @Scheduled(fixedRate = 60_000)  // 每 60 秒
    public void refreshCache() { ... }
}

// 注册到 services() 即可，框架自动发现 @Scheduled 方法
app.services(StatsService.class);
```

### 实现

- services() 实例化后扫描 @Scheduled 注解
- 内部创建 `ScheduledExecutorService`（单线程 daemon）
- cron 表达式用轻量 parser（5 字段：分 时 日 月 周）
- stop 时自动 shutdown scheduler

### 注解定义

```java
@Retention(RetentionPolicy.RUNTIME)
@Target(ElementType.METHOD)
public @interface Scheduled {
    String cron() default "";
    long fixedRate() default -1;    // ms
    long fixedDelay() default -1;   // ms
}
```

### AI 友好度

- 方法名即任务描述
- 不需要在 App.java 额外注册
- 出错时日志打印方法名 + 异常

## 不做

| 功能 | 原因 |
|------|------|
| @Cacheable | 引入缓存抽象过重，用户 Caffeine 3 行就够 |
| Event Bus | 小项目直接方法调用，大项目用 MQ |
| OpenAPI 生成 | 大功能，v0.7+ |
| WebSocket 房间 | 抽象过度，用户自己管连接更灵活 |
| Multitenancy / i18n / Metrics | 不在当前目标用户范围 |
