## 1. Auth 原语

### 设计原则

框架只定义一个抽象：**`ctx → userId`**。怎么取 token、怎么验证、用什么方案，全由用户决定。内置 JWT 只是默认实现。

### API 设计

```java
// 方式一：内置 JWT（最简场景，零依赖）
Aura.create()
    .jwt("your-secret-key")              // 默认 7 天过期
    .jwt("your-secret-key", 3600)        // 自定义过期秒数
    .routes(r -> {
        r.group("/api", api -> {
            api.before(Aura.requireAuth());   // 内置中间件
            api.get("/profile", userService, "profile");
        });
        r.post("/auth/login", authService, "login");
    })
    .start();

// 方式二：自定义认证（IAM / OAuth / Redis session / 任何方案）
Aura.create()
    .auth(ctx -> {
        // 用户决定从哪取 token（header、cookie、query param 都行）
        String token = ctx.cookie("session_token");
        // 用户决定怎么验证（调 IAM、查 Redis、解 OAuth...）
        Long userId = myIamClient.verify(token);
        return userId;  // 返回 null 表示未认证
    })
    .routes(r -> { ... })
    .start();

// 方式三：通过 Plugin 引入第三方方案
Aura.create()
    .plugin(new OAuthPlugin("client-id", "client-secret"))
    .start();

// Plugin 内部
public class OAuthPlugin implements AuraPlugin {
    public void install(Aura app) {
        app.auth(ctx -> {
            String token = ctx.header("X-Auth-Token");
            return oauthClient.verify(token);
        });
    }
}
```

### 框架提供的 API

| API | 作用 |
|-----|------|
| `app.auth(Function<Context, Long>)` | 注册认证函数（核心抽象） |
| `app.jwt(secret)` / `app.jwt(secret, expireSeconds)` | 注册内置 JWT 认证（调用 auth 注册默认实现） |
| `Aura.requireAuth()` | 返回 before 中间件，调用 auth 函数，失败返回 401 |
| `ctx.userId()` | 获取当前认证用户 ID（未认证时抛 IllegalStateException） |
| `app.signJwt(userId)` | 签发 JWT token（仅 jwt 模式可用） |

### 内置 JWT 行为

`app.jwt("secret")` 等价于：
```java
app.auth(ctx -> {
    String header = ctx.header("Authorization");
    if (header == null || !header.startsWith("Bearer ")) return null;
    return JwtSupport.verify(header.substring(7), secret);
});
```

- 算法：HMAC-SHA256
- Payload：`{"sub": "userId", "exp": timestamp}`
- 零外部依赖（Java 标准库 `javax.crypto.Mac` + `java.util.Base64` + fastjson2）

### `Aura.requireAuth()` 行为

- 调用注册的 auth 函数
- 返回非 null → 存入 ctx，继续执行
- 返回 null → 响应 401 `{"error": "Unauthorized"}` + abort
- 未注册 auth 函数时 → 抛 IllegalStateException（启动时报错，不等运行时）

### `ctx.userId()` 行为

- 返回 auth 函数存入的 userId（long）
- 未认证时抛 `IllegalStateException("Not authenticated — use Aura.requireAuth() middleware")`

### 环境变量

| 变量 | 作用 |
|------|------|
| `AURA_JWT_SECRET` | JWT 密钥 |
| `AURA_JWT_EXPIRE` | JWT 过期时间（秒，默认 604800 = 7天） |

### 不做的

- 不做 refresh token
- 不做 OAuth 协议实现（用户通过 plugin 接）
- 不做角色/权限体系
- 不做 token 黑名单

### 实现估算

- `aura-core/JwtSupport.java` — 签发 + 验证，~50 行
- `aura-core/Aura.java` — auth() / jwt() / signJwt() / requireAuth()，~30 行
- `aura-core/BaseContext.java` — userId() default 方法，~5 行
- 总计 ~85 行新增代码

## 2. JSON 格式 Access Log

### 问题

当前 access log：
```
GET /api/users → 200 (12ms)
```

对接 ELK/Loki/Grafana 时需要 JSON 格式。

### API 设计

```java
Aura.create()
    .accessLog(true)          // 当前行为：人类可读格式（保持不变）
    .accessLog("json")        // 新增：JSON 格式
    .start();

// 环境变量
AURA_ACCESS_LOG=json
```

### JSON 输出格式

```json
{"ts":"2024-01-01T10:00:00.123","method":"GET","path":"/api/users","status":200,"elapsed":12,"reqId":"a1b2c3d4e5f6","ip":"192.168.1.1"}
```

### 行为规则

- `accessLog(true)` 或 `accessLog("true")` — 人类可读格式（向后兼容）
- `accessLog("json")` — JSON 一行一条
- `accessLog(false)` 或不设 — 不输出（默认）
- 通过 SLF4J info 级别输出
- 包含字段：ts、method、path、status、elapsed（ms）、reqId、ip

### 影响范围

- `aura-core/Aura.java` — accessLog 字段从 boolean 改为 String
- `aura-web/UndertowStarter.java` — finally 块根据格式选择输出方式
- 约 15 行改动

## 3. CORS 细粒度配置

### 问题

当前 CORS 只有两种模式：
- `cors(true)` — Allow-Origin: `*`，全开
- `cors("https://example.com")` — 单个 origin

实际项目需要：
- 多个 origin（dev/staging/prod 不同域名）
- 自定义允许的 headers（如 `X-Request-Id`、`X-Auth-Token`）
- 是否允许 credentials（带 cookie 的跨域请求）

当前 Allow-Methods 和 Allow-Headers 写死在 UndertowStarter 里，无法配置。

### API 设计

```java
// 现有方式仍然可用（向后兼容）
app.cors(true);                         // 全开，Allow-Origin: *
app.cors("https://example.com");        // 单 origin

// 新增：细粒度配置
app.cors(cors -> cors
    .origins("https://app.com", "https://admin.app.com")  // 多 origin
    .headers("Content-Type", "Authorization", "X-Request-Id")  // 允许的请求头
    .credentials(true)  // Access-Control-Allow-Credentials: true
);
```

### 行为规则

- `origins(String...)` — 允许多个 origin。请求的 Origin header 匹配列表中任一个则放行，响应 Allow-Origin 设为请求的 Origin（不能用 `*` 因为有 credentials）
- `headers(String...)` — 覆盖默认的 Allow-Headers（默认 `Content-Type, Authorization`）
- `credentials(true)` — 添加 `Access-Control-Allow-Credentials: true`。此时 Allow-Origin 不能为 `*`，必须是具体域名
- Allow-Methods 保持 `GET, POST, PUT, DELETE, OPTIONS`（几乎不需要改）
- Max-Age 保持 86400（1 天）
- `cors(true)` 行为不变（向后兼容）

### 实现思路

新增 `CorsConfig` 类：

```java
public class CorsConfig {
    private List<String> origins = List.of("*");
    private String headers = "Content-Type, Authorization";
    private boolean credentials = false;

    public CorsConfig origins(String... origins) { this.origins = List.of(origins); return this; }
    public CorsConfig headers(String... headers) { this.headers = String.join(", ", headers); return this; }
    public CorsConfig credentials(boolean enabled) { this.credentials = enabled; return this; }
}
```

`Aura.java` 添加：
```java
public Aura cors(Consumer<CorsConfig> config) {
    CorsConfig c = new CorsConfig();
    config.accept(c);
    this.corsConfig = c;
    return this;
}
```

`UndertowStarter` dispatch 里根据 CorsConfig 动态设置 headers。

### 影响范围

- 新增 `aura-core/CorsConfig.java` — ~20 行
- `aura-core/Aura.java` — 添加 cors(Consumer) 重载 + corsConfig 字段
- `aura-web/UndertowStarter.java` — CORS header 逻辑改为读 CorsConfig
- 约 40 行改动

## 4. TestClient `resp.header()`

### 问题

TestClient.Response 当前只有 `status()`、`body()`、`json()`、`expect()`、`bodyContains()`。AI 写测试时无法验证响应头（X-Request-Id、Content-Type 等）。

### API 设计

```java
String reqId = client.get("/user/1").execute().header("X-Request-Id");
String ct = client.get("/user/1").execute().header("Content-Type");
```

### 实现

TestClient.Response 从 record 改为 class，增加 headers Map。MockContext 在设置响应头时收集到 Map 中，传给 Response。

### 影响范围

- `aura-web/TestClient.java` — Response 改造 + MockContext header 收集
- ~15 行

## 5. Row getter 默认值重载

### 问题

AI 每次取 Row 字段值都要写 null 判断：
```java
String nickname = row.getStr("nickname");
String display = nickname != null ? nickname : "";
```

### API 设计

```java
String display = row.getStr("nickname", "");
int age = row.getInt("age", 0);
long score = row.getLong("score", 0L);
boolean active = row.getBool("active", false);
```

### 实现

给 Row 现有的 4 个 getter 各加一个带默认值的重载：

```java
public String getStr(String key, String defaultValue) {
    String v = getStr(key);
    return v != null ? v : defaultValue;
}
public int getInt(String key, int defaultValue) {
    Integer v = getInt(key);
    return v != null ? v : defaultValue;
}
public long getLong(String key, long defaultValue) {
    Long v = getLong(key);
    return v != null ? v : defaultValue;
}
public boolean getBool(String key, boolean defaultValue) {
    Boolean v = getBool(key);
    return v != null ? v : defaultValue;
}
```

### 影响范围

- `aura-db/Row.java` — 4 个方法重载，~10 行
