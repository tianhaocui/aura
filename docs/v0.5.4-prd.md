## 1. app.props(prefix, Class) — 配置自动绑定

### 动机

目前取一组相关配置需要逐个调用 `app.prop(key)`，写起来重复且容易漏字段。提供自动绑定到 Record 的能力，一行搞定。

### API 设计

```java
// 配置文件 aura.properties
bnp.api.key=abc123
bnp.api.url=https://example.com
bnp.api.timeout=30

// Record 定义
record BnpConfig(String key, String url, int timeout) {}

// 使用
BnpConfig config = app.props("bnp.api.", BnpConfig.class);
// 自动去掉前缀，按字段名匹配
```

### 行为

- 去掉前缀后，按 Record 组件名（component name）匹配 properties key
- 类型转换规则：
  - String → 直接赋值
  - int/Integer → Integer.parseInt()
  - long/Long → Long.parseLong()
  - boolean/Boolean → "true".equalsIgnoreCase()
  - double/Double → Double.parseDouble()
- 匹配不到的字段：String 为 null，数值类型为 0，boolean 为 false
- 无匹配 key 时返回全空 Record（不抛异常）
- 仅支持 Record 类型（不支持普通 POJO）

### 示例

```java
// 多数据源配置
record DbConfig(String url, String username, String password, int poolSize) {}

DbConfig main = app.props("db.main.", DbConfig.class);
DbConfig analytics = app.props("db.analytics.", DbConfig.class);

// 第三方 API 配置
record WiseConfig(String authorization, String baseUrl) {}
WiseConfig wise = app.props("wise.v2.", WiseConfig.class);
```

## 4. 完整 HTTP 方法支持

### 动机

当前只支持 GET/POST/PUT/DELETE，缺少 PATCH（RESTful 部分更新）和 HEAD/OPTIONS 的路由注册能力。客户端团队看到 PUT 代替 PATCH 会困惑。

### 需要新增的方法

| 方法 | 用途 | 优先级 |
|------|------|--------|
| PATCH | RESTful 部分更新 | P0 — 开发者最常要求 |
| HEAD | 检查资源是否存在（无 body） | P1 — 自动基于 GET handler 生成 |
| OPTIONS | 手动控制 CORS preflight / API 发现 | P2 — 大部分场景已被 CORS 配置覆盖 |

### 改动范围

1. **BaseRouter** — 加 `patch()`、`head()`、`options()` 方法
2. **Aura** — 加 `app.patch()`、`app.head()`、`app.options()` 快捷方法
3. **UndertowStarter dispatch** — switch 加 `"PATCH"`、`"HEAD"`、`"OPTIONS"` 分支
4. **RouteEntry** — 已支持任意 method 字符串，无需改
5. **CORS Allow-Methods header** — 加上 `PATCH, HEAD`
6. **HEAD 自动处理**：如果注册了 GET handler 但没注册 HEAD，自动用 GET handler 执行但不发 body

### API

```java
// 路由注册
app.patch("/user/{id}", (int id, PatchReq req) -> update(id, req));

app.routes(r -> {
    r.patch("/item/{id}", itemService, "patch");
    r.head("/file/{id}", fileService, "exists");
});
```

## 5. JWT 支持 String subject

### 动机

内置 JWT 的 `sign(long userId)` + `verify()→Long` 硬编码了 long 类型，无法用 UUID 等字符串作用户 ID。生产项目被迫引入 jjwt 替代。

### API 设计

```java
// 新增 — String subject（通用）
String token = app.signJwt("uuid-or-any-string");
String subject = app.verifyJwt(token);  // 返回 String，null = 无效/过期

// 保留向后兼容 — long userId
String token = app.signJwt(123L);  // 内部调 signJwt(String.valueOf(123))
// verify 统一返回 String，调用方自行 Long.parseLong()
```

### 改动

- `JwtSupport.sign(String subject)` — payload 的 sub 字段直接存字符串
- `JwtSupport.verify(String token)` → 返回 `String`（原来返回 `Long`）
- `Aura.signJwt(long)` 保留，内部委托给新方法
- `Aura.signJwt(String)` 新增
- `Aura.verifyJwt(String token)` 新增，返回 String
- `ctx.userId()` 返回类型改为 `String`（**breaking change**，需要迁移说明）

### 迁移

```java
// 之前
long uid = ctx.userId();
// 之后
String uid = ctx.userId();
long uidLong = Long.parseLong(ctx.userId()); // 如果确实是数字
```

## 6. Row 保留原始大小写

### 动机

当前 `Row.set(key, value)` 强制 `key.toLowerCase()`，导致 PG 返回的 `camelCase` alias 全部丢失大小写。生产项目被迫写 150 行结果转换层。

### 方案

去掉 `set()` 里的 `.toLowerCase()`，`get()` 做 case-insensitive fallback：

```java
public Object get(Object key) {
    // 先精确匹配
    Object v = super.get(key);
    if (v != null || super.containsKey(key)) return v;
    // fallback: case-insensitive
    if (key instanceof String k) {
        for (var entry : entrySet()) {
            if (k.equalsIgnoreCase(entry.getKey())) return entry.getValue();
        }
    }
    return null;
}
```

### 行为变化

- `row.set("userName", "alice")` → key 保留为 `userName`（不再变成 `username`）
- `row.get("username")` 仍然能匹配到 `userName`（case-insensitive fallback）
- DB 查询返回的列名保持原始大小写（如 `SELECT u.name AS userName` → key = `userName`）

### Breaking Change 风险

依赖 `row.get("lowercase_key")` 访问原本是 `CamelCase` 的字段的代码需要调整。但由于 fallback 机制，大部分场景向后兼容。

## 7. TestClient 增强

### 动机

TestClient 缺少对新 HTTP 方法的支持，且常见测试场景（登录→带 token 请求、JSON 断言）写起来繁琐。

### 7.1 补全 HTTP 方法

```java
client.patch("/user/1").body(Map.of("name", "new")).expect(200);
client.head("/file/1").expect(200);
client.options("/api/items").expect(204);
```

改动：`TestClient` 加 `patch()`、`head()`、`options()` + `TestClient.of()` 里的 switch 补对应 case。

### 7.2 TestSession — 自动 token 管理

```java
// 登录拿 token，后续请求自动附加
TestSession session = client.session();
session.login(client.post("/login").body(creds).execute());  // 自动提取 token
session.get("/api/me").expect(200);  // Authorization header 自动附加

// 或者手动指定
TestSession session = client.session("Bearer " + token);
session.get("/api/me").expect(200);
```

实现：`TestSession` 包装 `TestClient`，持有一个 default header map，每次 request 自动 merge。

### 7.3 expectJson — JSONPath 断言

```java
client.get("/user/1").execute()
    .expect(200)
    .expectJson("$.name", "alice")
    .expectJson("$.age", 30)
    .expectJson("$.address.city", "London");
```

实现：用 fastjson2 的 JSONPath（已在依赖中），无需引入新依赖。

```java
// Response 里加
public Response expectJson(String jsonPath, Object expected) {
    Object actual = com.alibaba.fastjson2.JSONPath.eval(
        com.alibaba.fastjson2.JSON.parse(body), jsonPath);
    if (!java.util.Objects.equals(expected, actual)) {
        throw new AssertionError("JSONPath " + jsonPath + ": expected " + expected + " but got " + actual);
    }
    return this;
}
```

### 7.4 公开 MockContext

```java
// 从 package-private 改为 public — 允许用户直接构造
public class MockContext extends Context { ... }
```

用途：单元测试中直接构造 context 传给 handler，不经过路由匹配。

```java
var ctx = new MockContext(Map.of("id", "1"), Map.of(), Map.of(), null, app);
userService.get(ctx);
assertEquals(200, ctx.statusCode());
```

### 7.5 query 快捷方法

```java
client.get("/items").query("page", "1").query("size", "20").expect(200);
// 等价于 client.get("/items?page=1&size=20") 但更清晰
```

### 不纳入本版

| 功能 | 原因 |
|------|------|
| multipart/file upload | MockContext.file() 需要较大改动，v0.6 |
| cookie 自动管理 | 和 TestSession 合并考虑，v0.6 |
| SSE 断言增强 | 当前 mock 已能返回 SSE body，够用 |

## 8. Dev 热重载（aura-dev 模块）

### 动机

改一行代码 → mvn compile → 重启 JVM → 等连接池初始化 → 验证。整个循环 8-12 秒。Spring Boot 有 DevTools，Aura 需要同等能力。

### 架构：双 ClassLoader 热替换

```
Base ClassLoader (常驻，不重载)         User ClassLoader (丢弃重建)
├── Undertow (端口绑定保持)             ├── 用户 Service 类
├── HikariCP (连接池保持)               ├── 用户 Handler / Lambda
├── Aura 框架类                         ├── Record DTO
├── aura-core / aura-web / aura-db      └── 用户 main() 所在类
└── 第三方依赖 JAR
```

### 使用方式

```java
// 方式 1：启动参数（推荐）
java -jar app.jar --dev

// 方式 2：环境变量
AURA_ENV=dev java -jar app.jar

// 方式 3：代码
Aura.create().dev(true).start();
```

```xml
<!-- 仅开发期引入 -->
<dependency>
    <groupId>io.github.tianhaocui</groupId>
    <artifactId>aura-dev</artifactId>
    <version>0.5.4</version>
    <scope>provided</scope>
</dependency>
```

### 核心机制：重新执行 main()（方案 B）

用户代码零改动。reload 时重新调用用户的 main()，框架内部判断 reload 模式，复用基础设施：

```java
// 用户代码 — 完全不需要改
public static void main(String[] args) {
    Aura app = Aura.create();
    Db db = Db.create(app.prop("db.url"), app.prop("db.user"), app.prop("db.password"));
    app.register(db);
    app.service(new UserService(db), new OrderService(db));
    app.routes(r -> {
        r.get("/health", () -> "ok");
    });
    app.start();
}
```

Reload 模式下各方法的行为：

| 方法 | 正常启动 | Reload 模式 |
|------|---------|------------|
| `Aura.create()` | 新实例 | 返回已有实例，清空 routes/services/directRoutes |
| `Db.create(url, ...)` | 新连接池 | URL 匹配已有池则复用（幂等），避免重建 HikariCP |
| `app.register(x)` | 注册 bean | 同类型已存在则跳过 |
| `app.service(...)` | 注册 service | 用新 ClassLoader 的 class 重建实例 |
| `app.routes(r -> {...})` | 保存 lambda | 保存新 lambda（来自新 ClassLoader） |
| `app.start()` | 启动 Undertow + 编译路由 | 只重编译路由表，原子替换 |

### 实现细节

#### 8.1 Aura reload 入口

```java

private static volatile Aura RELOAD_INSTANCE;
private boolean reloadMode = false;

public static Aura create() {
    if (RELOAD_INSTANCE != null) {
        Aura app = RELOAD_INSTANCE;
        app.reloadMode = true;
        app.directRoutes.clear();
        app.services.clear();
        app.routeConfig = null;
        return app;

    return new Aura();

public void start() {

        reloadMode = false;
        starter.reloadRoutes(this);  // 只重编译路由
        return;
    }
    // ... 正常启动逻辑
}
```

#### 8.3 UndertowStarter 路由热替换

```java
// UndertowStarter.java
private volatile List<CompiledRoute> compiledRoutes;      // 改为 volatile
private volatile List<CompiledWsRoute> compiledWsRoutes;  // 改为 volatile

public void reloadRoutes(Aura app) {
    long t0 = System.currentTimeMillis();
    Router newRouter = new Router();

    for (var entry : app.directRoutes()) {
        BaseHandler handler = entry.handler() instanceof BaseHandler h
            ? h : new LambdaHandler(entry.handler());
        switch (entry.method()) {
            case "GET" -> newRouter.get(entry.path(), handler);
            case "POST" -> newRouter.post(entry.path(), handler);
            case "PUT" -> newRouter.put(entry.path(), handler);
            case "PATCH" -> newRouter.patch(entry.path(), handler);
            case "DELETE" -> newRouter.delete(entry.path(), handler);
            case "HEAD" -> newRouter.head(entry.path(), handler);
            case "OPTIONS" -> newRouter.options(entry.path(), handler);
        }
    }

    @SuppressWarnings("unchecked")
    var config = (Consumer<BaseRouter>) app.routeConfig();
    if (config != null) config.accept(newRouter);

    for (Object service : app.services()) {
        ServiceRegistrar.register(service, newRouter);
    }

    var newCompiled = compile(newRouter, "",
        new ArrayList<>(app.beforeHandlers()), new ArrayList<>(app.afterHandlers()));
    newCompiled.sort(...);
    detectDuplicateRoutes(newCompiled);

    this.compiledRoutes = newCompiled;  // 原子替换（volatile write）
    this.compiledWsRoutes = compileWsRoutes(newRouter);

    log.info("[aura-dev] routes reloaded in {}ms", System.currentTimeMillis() - t0);
}
```

#### 8.4 DevReloader — 文件监听 + 编译 + 触发 reload

```java
// aura-dev 模块核心类
public class DevReloader {

    private final Aura app;
    private final String mainClass;
    private final Path sourceDir;
    private final Path outputDir;
    private volatile URLClassLoader userClassLoader;
    private final String classpath;

    public DevReloader(Aura app, String mainClass) {
        this.app = app;
        this.mainClass = mainClass;
        this.sourceDir = Paths.get("src/main/java");
        this.outputDir = Paths.get("target/classes");
        this.classpath = System.getProperty("java.class.path");
    }

    public void start() {
        // 启动时检测：必须是 JDK（不是 JRE）
        if (ToolProvider.getSystemJavaCompiler() == null) {
            throw new IllegalStateException(
                "[aura-dev] requires JDK (not JRE). javax.tools.JavaCompiler not available.");
        }

        Aura.RELOAD_INSTANCE = app;  // 开启 reload 模式
        Thread watcher = new Thread(this::watchLoop, "aura-dev-watcher");
        watcher.setDaemon(true);
        watcher.start();
        log.info("[aura-dev] watching {} for changes", sourceDir);
    }

    private void watchLoop() {
        try (WatchService ws = FileSystems.getDefault().newWatchService()) {
            registerRecursive(sourceDir, ws);
            while (true) {
                WatchKey key = ws.take();
                // debounce: 收集 200ms 内所有变更
                Thread.sleep(200);
                List<Path> changed = collectChangedFiles(key);
                key.reset();
                if (!changed.isEmpty()) {
                    reload(changed);
                }
            }
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        } catch (Exception e) {
            log.error("[aura-dev] watcher error", e);
        }
    }

    private void reload(List<Path> changedFiles) {
        long t0 = System.currentTimeMillis();
        try {
            // 1. 增量编译（javax.tools.JavaCompiler）
            boolean compiled = compile(changedFiles);
            if (!compiled) {
                log.warn("[aura-dev] compilation failed, skipping reload");
                return;
            }

            // 2. 关闭旧 ClassLoader
            if (userClassLoader != null) {
                userClassLoader.close();
            }

            // 3. 新 ClassLoader（父=当前线程的 CL，即 Base CL）
            userClassLoader = new URLClassLoader(
                new URL[]{ outputDir.toUri().toURL() },
                getClass().getClassLoader()
            );

            // 4. 用新 ClassLoader 重新调用 main()
            Class<?> mainCls = userClassLoader.loadClass(mainClass);
            Method main = mainCls.getMethod("main", String[].class);
            main.invoke(null, (Object) new String[]{}); // 触发 Aura reload 流程

            long elapsed = System.currentTimeMillis() - t0;
            log.info("[aura-dev] reloaded in {}ms ({} files)", elapsed, changedFiles.size());
        } catch (Exception e) {
            log.error("[aura-dev] reload failed: {}", e.getMessage());
            // 失败不影响当前运行的路由 — 旧路由仍然服务请求
        }
    }

    private boolean compile(List<Path> files) {
        JavaCompiler compiler = ToolProvider.getSystemJavaCompiler();
        DiagnosticCollector<JavaFileObject> diagnostics = new DiagnosticCollector<>();
        try (StandardJavaFileManager fm = compiler.getStandardFileManager(diagnostics, null, null)) {
            var sources = fm.getJavaFileObjectsFromPaths(files);
            var options = List.of(
                "-cp", classpath,
                "-d", outputDir.toString(),
                "-parameters",                // Aura 必需
                "-source", "17",
                "-target", "17"
            );
            boolean success = compiler.getTask(null, fm, diagnostics, options, null, sources).call();
            if (!success) {
                for (var d : diagnostics.getDiagnostics()) {
                    log.error("[aura-dev] {}:{} {}", 
                        d.getSource().getName(), d.getLineNumber(), d.getMessage(null));
                }
            }
            return success;
        } catch (Exception e) {
            log.error("[aura-dev] compile error", e);
            return false;
        }
    }

    private void registerRecursive(Path root, WatchService ws) throws IOException {
        Files.walkFileTree(root, new SimpleFileVisitor<>() {
            @Override
            public FileVisitResult preVisitDirectory(Path dir, BasicFileAttributes attrs) throws IOException {
                dir.register(ws, StandardWatchEventKinds.ENTRY_MODIFY, StandardWatchEventKinds.ENTRY_CREATE);
                return FileVisitResult.CONTINUE;
            }
        });
    }

    private List<Path> collectChangedFiles(WatchKey key) {
        List<Path> files = new ArrayList<>();
        for (WatchEvent<?> event : key.pollEvents()) {
            Path changed = ((Path) key.watchable()).resolve((Path) event.context());
            if (changed.toString().endsWith(".java")) {
                files.add(changed);
            }
        }
        // drain 额外的 pending keys (debounce 期间的)
        WatchKey extra;
        while ((extra = key.pollEvents() != null ? null : null) != null) {} // simplified
        return files;
    }
}
```

#### 8.5 Cross-ClassLoader 安全性

| 场景 | 是否安全 | 原因 |
|------|---------|------|
| 新 Service 引用 Db | ✅ | Db 在父 CL，子 CL 可见 |
| 新 Service 引用 Aura | ✅ | Aura 在父 CL |
| Handler 返回 Record DTO | ✅ | DTO 在子 CL，序列化用反射（父 CL 的 fastjson2 能处理） |
| 旧线程引用旧 Service | ⚠️ | in-flight 请求可能用旧 handler，下个请求用新的 |

in-flight 请求安全性：volatile 替换路由表后，正在执行的请求继续用旧 handler 完成（引用不会断），新请求用新 handler。天然安全。

#### 8.6 ClassLoader 泄漏防护

```java
// reload 前清理
private void cleanupBeforeReload() {
    // 取消用户注册的 ScheduledFuture（如果有）
    app.fireReloadCleanup();
}
```

用户可选 hook：
```java
app.onReload(() -> {
    // 清理 Timer、ScheduledExecutor 等
    myScheduler.shutdown();
});
```

大部分项目不需要这个 hook（dev mode 不追求零泄漏，JVM 最终会回收）。

#### 8.7 降级策略

如果 ClassLoader 热替换失败（编译错误、类加载异常），**不影响当前运行的服务**：

```
[aura-dev] file changed: UserService.java
[aura-dev] compilation failed:
  UserService.java:25 cannot find symbol: method foo()
[aura-dev] reload skipped, current routes still serving
```

编译成功但 reload 异常时：
```
[aura-dev] reload failed: NoSuchMethodException ...
[aura-dev] current routes still serving (fix the error and save again)
```

#### 8.9 连接池 lazy init（dev mode 配套）

Dev mode 下 HikariCP 不在启动时建连接，加速首次启动：

```java
// Db.create() 在 dev mode 时自动设置
if (Aura.isDevMode()) {
    config.setInitializationFailTimeout(-1);  // 不等待首个连接验证
    config.setMinimumIdle(0);                 // 不预热空闲连接
}
```

首次查询时才建连接，启动快 1-3s。生产模式不受影响。

### 预期性能

| 阶段 | 耗时 |
|------|------|
| WatchService 检测变更 | ~50ms |
| Debounce 等待 | 200ms |
| javax.tools.JavaCompiler 增量编译 | ~300-800ms |
| ClassLoader 新建 + main() 重执行 | ~50-100ms |
| 路由重编译 + volatile 替换 | ~30ms |
| **总计** | **~0.6-1.2s** |

对比：
- Spring Boot DevTools: ~1.5-2s（需要重建 ApplicationContext）
- 当前 Aura 手动重启: ~8-12s
- **Aura dev mode: <1.5s**

### 限制（与 Spring DevTools 相同）

| 限制 | 原因 | 缓解 |
|------|------|------|
| 新加/删除第三方依赖需全重启 | 依赖在 Base ClassLoader | 检测 pom.xml 变更时提示用户重启 |
| static 字段会重置 | 新 ClassLoader = 新 Class | 状态放 app.register() 里，不放 static |
| 改 aura.properties 需全重启 | 配置在 Aura 实例里加载 | 可增强：监听 .properties 变更也触发 reload |
| 需要 JDK 运行 | javax.tools.JavaCompiler | 启动时检测并报错 |

### 模块结构

```
aura-dev/
├── src/main/java/io/aura/dev/
│   ├── DevReloader.java       // 核心：监听 + 编译 + reload
│   ├── IncrementalCompiler.java   // javax.tools 封装

└── pom.xml
```

通过 SPI 自动发现：用户加了 aura-dev 依赖 + dev mode 开启，自动激活。无需额外配置。

### 9.3 事务支持 checked exception

当前 `transaction(Supplier<T>)` 的 Supplier.get() 不声明 throws，导致事务内无法调用 `bodyOrThrow()` 等声明了 checked exception 的方法。

```java
// 新增函数接口
@FunctionalInterface
public interface ThrowingSupplier<T> {
    T get() throws Exception;
}

// 新增重载
public <T> T transaction(ThrowingSupplier<T> block) throws Exception {
    // ... 和原有逻辑相同，但允许异常传播
}

public void transactionThrows(ThrowingRunnable block) throws Exception {
    transaction(() -> { block.run(); return null; });
}
```

## 10. bodyOrThrow 结构化错误

### 动机

校验失败时返回 `"name: must not be blank; age: must be >= 0"` 分号拼接字符串，不是结构化 JSON。前端无法按字段显示错误。

### 改进

```java
// 之前的响应
{"error": "name: must not be blank; age: must be >= 0"}

// 改进后
{
  "error": "Validation failed",
  "errors": [
    {"field": "name", "message": "must not be blank"},
    {"field": "age", "message": "must be >= 0"}
  ]
}
```

### 实现

BeanValidator.validate() 改为抛出包含结构化信息的异常：

```java
public record FieldError(String field, String message) {}

public static class ValidationException extends RuntimeException {
    private final List<FieldError> errors;

    public ValidationException(List<FieldError> errors) {
        super("Validation failed");
        this.errors = errors;
    }

    public List<FieldError> errors() { return errors; }
}
```

框架默认异常 handler 捕获 ValidationException 时序列化为结构化 JSON。
